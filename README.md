# Java Console Calculator App

A simple Java console calculator application supporting both interactive and command-line modes, with integrated Sentry error tracking and session logging. The project is built with Gradle and is suitable for quick experimentation, learning, and demonstration of error monitoring in Java applications.

## Description

This project is a Java console calculator that can be run in two ways:
- **Interactive Mode:** Users are prompted for input in the console.
- **Command-Line Mode:** Users can pass calculation arguments directly via the command line.

All errors are tracked using [Sentry](https://sentry.io/), and session logs are stored locally for debugging and auditing.

---

## Tech Stack

- **Java** (version 11+ recommended)
- **Gradle** (build automation)
- **Sentry Java SDK** (`io.sentry:sentry:7.7.0`) for error tracking
- **H2 Database** (`com.h2database:h2:2.2.224`) — in-memory database for operation history
- **JUnit 5** (`org.junit.jupiter:junit-jupiter:5.10.2`) for unit testing
- **IntelliJ IDEA** (optional, for development; `.iml` project file included)
- **Bash** (for `run.sh` script, optional)

## Installation & Setup

1. **Clone the repository:**
   ```sh
   git clone <repo-url>
   cd learn-gh-copilot/java-console-app
   ```

2. **Build the project using Gradle:**
   ```sh
   gradle build
   ```
   _Or, if using the Gradle Wrapper (if present):_
   ```sh

   ```

   This will download dependencies and compile the source code.

## Running the Project

You can run the calculator in two ways: interactively or by passing command-line arguments.

### 1. Interactive Mode

Start the calculator and follow the prompts:

**Using Gradle:**
```sh
gradle run
```
_Or with the Gradle Wrapper:_
```sh
./gradlew run
```

**Using the shell script:**
```sh
./run.sh
```

### 2. Command-Line Arguments Mode

You can pass calculation arguments directly (e.g., `2 + 2`):

**Using Gradle:**
```sh
gradle run --args="2 + 2"
```

**Using the shell script:**
```sh
./run.sh 2 + 2
```

_The result will be printed to the console. Errors (such as invalid input) will be logged and reported to Sentry._

### 3. View Calculation History

To view all past calculations from the current session:

**Using the shell script:**
```sh
./run.sh history
```

**Using Gradle:**
```sh
gradle run --args="history"
```

## Operation History

Every calculation performed is automatically saved to an **H2 in-memory database**. The history feature records:

- **Operands** (`num1`, `num2`)
- **Operator** (`+`, `-`, `*`, `/`)
- **Result**
- **Timestamp** (when the calculation was performed)

To view the history, pass `history` as a command-line argument (see [View Calculation History](#3-view-calculation-history) above).

> **Note:** The H2 database runs in-memory only (`jdbc:h2:mem:calculator`). All history data is lost when the application exits. This is by design for simplicity and quick experimentation.

## Running Tests

Unit tests are written with **JUnit 5**. To run all tests:

```sh
cd java-console-app
gradle test
```

Test reports are generated in `java-console-app/build/reports/tests/test/index.html`.

## Logging & Error Tracking

- **logs/**: All session logs are saved in this directory for debugging and auditing.
- **Sentry Integration**: All unhandled exceptions and errors are automatically reported to Sentry using the Sentry Java SDK (`io.sentry:sentry:7.7.0`).  
  _To configure Sentry (e.g., set your DSN), update the Sentry initialization in `Main.java` or provide the appropriate environment variables._
